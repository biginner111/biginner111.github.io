进程的概念

进程：程序执行时候的一个实例。
1.程序是被存储在磁盘上，是机器指令和数据文件；
2.内核将程序加载到内存并且被cpu执行时就形成了进程；
3.一个程序可以运行多个进程；
4.通常将进程称为任务；
5.认为进程是承载资源的实体（cpu运行时间，内存资源）。

相关命令

pstree以树形图的方式显示进程关系


ps以表格的方式显示进程
ps显示当前用户拥有的控制终端信息

-a显示所有用户拥有的控制终端进程信息

-x显示没有控制终端的进程信息

-u更详细

-w更大的宽度显示

进程信息列表：
USER :进程用户id
PID ：进程pid
%CPU ：cpu利用率
%MEM ：内存利用率
VSZ ：占用虚拟内存大小（kb）
RSS ：占用物理内存大小（kb）
TTY ：终端次设备号
STAT ：进程状态
START ：起始时间
TIME ：运行时间
COMMAND ：运行命令

进程状态STAT：

< 表示高优先级的进程(not nice to other users)

N 低优先级(nice to other users)

+ 属于某个前台组的进程

s 某一个回话的leader进程

l 线程加锁

+ 后台运行进程组

D 不可中断睡眠 (通常是在IO操作) 收到信号不唤醒和不可运行, 进程必须等待直到有中断发生

R 正在运行或可运行（在运行队列排队中）

S 可中断睡眠 (休眠中, 受阻, 在等待某个条件的形成或接受到信号)

T 已停止的 进程收到SIGSTOP, SIGSTP, SIGTIN, SIGTOU信号后停止运行

W 正在换页(2.6.内核之前有效)

X 死进程 (未开启)

Z 僵尸进程 进程已终止, 但进程描述符存在, 直到父进程调用

父孤子尸

  1. 进程之间存在可以存在父子关系，一个进程可以有多个子进程，是一个进程只能有一个父进程。整个系统有一个根进程，即0号进程；
  2. 内核中有一个孤儿院进程，用于收养其父亲进程先于子进程结束的子进程；
  3. 父进程先于子进程结束，子进程成为孤儿进程，被孤儿院进程收养；
  4. 子进程先于父进程一步结束，但是父进程没有回收子进程尸体（终止状态信息），子进程变为僵尸进程，僵尸进程将占用内存空间。

进程标识

  1. 每个进程都有一个非负整数编号，即pid；
  2. 进程的pid是独一无二的，进程结束以后可以被新创建的进程使用；
  3. 0号进程叫做调度进程也叫交换进程，是所有进程的根进程，是系统内核的一部分，在磁盘中找不到他；
  4. 1号进程，即init进程，系统完成自举以后，读写与系统初始化相关的文件，进行系统初始化，引导系统进入特定的状态，以超级用户特权运行，永不终止；
  5. 除了根进程以外，所有进程都有父进程，其pid是子进程的ppid。

相关查询进程id函数：
#include <unistd.h>
getpid();//获取进程pid
getppid();//获取进程ppid
getuid();//获取进程实际用户id
getgid();//获取进程实际组id
geteuid();//获取进程有效用户id
getegid();//夺取进程有效组id

创建子进程

功能：创建调用该函数的进程的子进程。
返回值：失败返回-1，成功在父进程返回子进程pid，在子进程返回0。
系统中总的线程数目达到上限或者进程数目达到上限都会失败。

父子进程关系

由fork产生的子进程是父进程的不完全副本，子进程会复制除了代码以外的存储区（堆栈，数据区，bss等）内容；
子进程和父进程共用一块代码；
子进程还会复制父进程的文件描述符表，但是不会复制文件表项，i结点，锁表，v结点等。

一、文件的读写与冲突

当多个进程同时对同一个文件进行读取或者写入就会造成文件的内容是不确定的。我们已经知道读取或者写入文件都会改变文件的读取写入位置。

二、避免冲突的方法

我们可以制定一定的规则来解决该问题。 制定的规则如下：

1.当有进程对文件某区域进行写入操作时，避免其他进程对该文件该区域的写入和读取；

2.当有进程对该文件进行读取时，应该避免其他进程对该文件的写入，但是读取并不影响。

这个规则叫做文件锁。具体概念为： 1.将文件锁分为读锁和写锁。

2.在文件的特定位置只能够添加一把写锁，写完成以后，进行解锁才可以进行下一次的读写操作。

3.文件的特定位置能够添加多把读锁，添加读锁后不能添加写锁，全部读锁解开以后可以添加。

4.读写完成以后要进行解锁。

5.一个进程要进行写入和读取时才在文件上进行加锁，且要满足加锁条件才能进行加锁，不满足的时候根据锁的阻塞类型会进行阻塞等待。

三、文件锁

使用函数:fcntl

#include <fcntl.h>
int fcntl(int fd,F_SETLK/F_SETLTK,struct flock*lock);

功能：加解锁。

输入参数：

fd：文件描述符

F_SETLK(以非阻塞方式加锁)/F_SETLTK(以阻塞方式加锁)

struct flock{

short l_type;//锁的类型:F_RDLCK(读锁)、F_WRLCK(写锁)、F_UNLCK(解锁)

   short l_whence;//偏移的起点:SEEK_SET、SEEK_CUR、SEEK_END 

 	off_t l_start; //锁区偏移，从l_whence

    off_t l_len;  //锁区长度(字节)

    pid_t l_pid; //阻塞我们加锁的进程ID

}

返回值：成功0,失败-1。

注意：

1.同一进程的不同线程之间是起不了文件锁的作用的，只能在进程间其作用；

2.当使用close关闭文件时，加在该进程的该文件的锁也解除；

3.当进程关闭时，该进程中所打开的锁全部结束；

4.使用fork/fork创建的子进程不会继承父进程的文件锁；

5.当文件的文件描述符没有FD_CLOEXEC时，使用exec创建的子进程会继承父亲的文件锁。

文件锁其作用的条件为：各个进程对文件进行操作，严格按照先加锁在进行操作，操作完毕，解除文件锁，文件锁才能起到文件锁的作用。如果用户不按照文件锁的使用规则进行操作，文件锁也发挥不了作用。

四、文件锁的内核结构

1.系统内核会收集用户所加锁的内容flock组成一张锁表，当加锁时就会去查锁表，当有冲突时就会报错或者阻塞。

2.系统内核会把锁表起始地址存储在该文件的v结点点中。

文件描述符表存储文件表项指针，文件表项指针指向文件表项，文件表项存储v结点指针，v结点指针指向v结点，v结点存储锁表的指针锁表指针指向锁表的头。

存储内容：

文件描述符表：文件表项指针，文件描述符为一条条存储在文件描述符表的下标。

文件表项:v结点指针，文件读写位置，文件状态标志。

v结点：锁表指针，i结点指针，v结点信息。

锁表：锁结点链式连接构成。

锁结点：类型，偏移，锁区大小，进程pid，下一个结点指针

五、文件的元数据

#include<sys/stat.h>
int stat(char const path,struct statbuf);
int fstat(int fd,struct stat*buf);
int lstat();char const path,struct statbuf

功能：从i结点中提取文件元数据内容信息。

参数： path文件路径

fd文件描述符

buf文件元数据信息 返回值：成功0,失败-1 lstat不跟踪符号连接，例如如果a.txt,b.txt建立了符号链接，查看a.txt的元数据就是b.txt的

struct stat {
dev_t st_dev; /* 包含文件的设备的 ID /
ino_t st_ino; / Inode 编号 /
mode_t st_mode; / 文件类型和模式 /
nlink_t st_nlink; / 硬链接数 /
uid_t st_uid; / 文件所有者的用户 ID /
gid_t st_gid; / 文件所有者的组 ID /
dev_t st_rdev; / 设备 ID（如果是特殊文件） /

blkcnt_t st_blocks; / 分配给文件的 512B 块的数量 */

struct timespec  st_atim;  /* 最后访问时间 */
struct timespec  st_mtim;  /* 最后修改时间 */
struct timespec  st_ctim;  /* 最后状态更改时间 */

#define st_atime  st_atim.tv_sec  /* 向后兼容性 */
#define st_mtime  st_mtim.tv_sec
#define st_ctime  st_ctim.tv_sec

};

mode_t:

在32位系统中大小为32位，目前只有后十六位有效。分别代表：

B15-B12：文件类型

文件类型有：普通文件，目录文件，本地套接字文件，字符设备，块设备，符号链接，有名管道，掩码

B11-B9：设置用户id，组id，粘滞位

B8-B6：拥有者的读写执行

B5-B3：拥有者组的读写执行

B2-B0：其他组成员的读写执行

通过查询到的mode_t和一些宏可以判断文件类型

S_ISLNK(st_mode):是否是一个连接.
S_ISREG是否是一个常规文件.
S_ISDIR是否是一个目录
S_ISCHR是否是一个字符设备.
S_ISBLK是否是一个块设备
S_ISFIFO是否是一个FIFO文件.
S_ISSOCK是否是一个SOCKET文件.

六、内存映射文件

一、为什么要进行映射

1、将一个普通文件映射到内存中，通常在需要对文件进行频繁读写时使用，这样用内存读写取代I/O读写，以获得较高的性能；
2、将特殊文件进行匿名内存映射，可以为关联进程提供共享内存空间；
3、为无关联的进程提供共享内存空间，一般也是将一个普通文件映射到内存中。

二、对应函数

#include <sys/mman.h>
void* mmap(void* start,size_t length,int port,int flags,int fd,off_t offset);//建立映射

功能：建立虚拟内存到物理内存或者磁盘的映射。

输入：

start：映射到虚拟内存的起始地址，填写NULL自动选择

length：映射区字节大小，自动按页元整

port：映射区的操作权限

PORT_READ:映射区可读

PORT_WRITE：映射区可写

PORT_EXEC：映射区可执行

PORT_NONE：映射区不可访问

flags:

MAP_FIXED 如果参数start所指的地址无法成功建立映射时，则放弃映射，不对地址做修正

MAP_SHARED对映射区域的写入数据会复制回文件内。

MAP_PRIVATE 对映射区域的写入操作只会反映到缓冲区，而不会写入到文件。

MAP_ANONYMOUS建立匿名映射。此时会忽略参数fd，不涉及文件，而且映射区域无法和其他进程共享。

MAP_DENYWRITE只允许对映射区域的写入操作，其他对文件直接写入的操作将会被拒绝。

MAP_LOCKED 将映射区域锁定住，这表示该区域不会被置换（swap）。

fd:文件描述符

offset：偏移量，自动按页元整（4096）

输出：成功返回映射区虚拟内存的起始地址，失败返回MAP_FALID(-1)。

int munmap(void* start,size_t length);//解除映射

功能：解除虚拟内存到物理内存或者是磁盘的映射

输入：

start：映射区的开始地址。
length：映射区的长度。

输出：成功0,失败-1.

三、实例

#include <stdio.h>
#include <sys/mman.h>//mmap,munmap
#include <fcntl.h>//open
#include <errno.h>//errno
#include <sys/stat.h>//stat,fstat,lstat
#include //close
int main(int argc, char *argv[])
{
int fd = 0;
char *ptr = NULL;
struct stat buf = {0};

if (argc < 2)
{
    printf("please enter a file!\n");
    return -1;
}

if ((fd = open(argv[1], O_RDWR)) < 0)
{
    printf("open file error\n");
    return -1;
}

if (fstat(fd, &buf) < 0)
{
    printf("get file state error:%d\n", errno);
    close(fd);
    return -1;
}

ptr = (char *)mmap(NULL, buf.st_size, PROT_READ|PROT_WRITE, MAP_SHARED, fd, 0);
if (ptr == MAP_FAILED)
{
    printf("mmap failed\n");
    close(fd);
    return -1;
}

close(fd);

printf("length of the file is : %d\n", buf.st_size);
printf("the %s content is : %s\n", argv[1], ptr);

ptr[3] = 'a';
printf("the %s new content is : %s\n", argv[1], ptr);
munmap(ptr, buf.st_size);
return 0;

}
